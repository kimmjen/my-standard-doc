# JSON (JavaScript Object Notation)

[RFC 8259](https://datatracker.ietf.org/doc/html/rfc8259)는 인터넷에서 가장 널리 쓰이는 데이터 교환 포맷인 **JSON**의 문법을 정의한 표준입니다.

개발자가 흔히 겪는 파싱 에러나 데이터 호환성 문제는 대부분 이 표준의 사소한 규칙을 어겼기 때문에 발생합니다.

---

## 1. 왜 표준을 알아야 할까요?

*   **"키(Key)에 따옴표가 없어도 되나요?"**: 자바스크립트 객체에서는 되지만, **JSON 표준에서는 반드시 큰따옴표(`"`)를 써야 합니다.** 이를 어기면 다른 언어(Python, Java 등)의 파서에서 에러가 납니다.
*   **"주석(Comment)을 달아도 되나요?"**: 표준 JSON은 **주석을 지원하지 않습니다.** 설정 파일로 JSON을 쓸 때 가장 많이 하는 실수입니다. (그래서 VS Code는 'JSON with Comments'라는 비표준 모드를 따로 지원합니다.)

---

## 2. 핵심 문법 규칙

1.  **문자열 (Strings)**: 반드시 **큰따옴표(`"`)**로 감싸야 합니다. 작은따옴표(`'`)는 허용되지 않습니다.
2.  **숫자 (Numbers)**: 10진수만 허용됩니다. `NaN`이나 `Infinity`는 JSON 표준에 없습니다. (이 값들을 보내면 `null`로 변환되거나 에러가 납니다.)
3.  **마지막 쉼표 (Trailing Comma)**: 배열이나 객체의 마지막 항목 뒤에 **쉼표를 찍으면 안 됩니다.** (오래된 브라우저나 엄격한 파서에서 에러 원인 1순위)

---

## 3. 데이터 타입

JSON이 표현할 수 있는 값은 딱 6가지입니다.
*   `string`
*   `number`
*   `boolean` (`true`, `false` - 소문자만)
*   `null`
*   `object` ( `{ ... }` )
*   `array` ( `[ ... ]` )

> [!WARNING]
> 날짜(Date) 타입은 JSON 표준에 없습니다.
> 그래서 보통 ISO 8601 문자열(`"2024-05-21T10:00:00Z"`)로 변환해서 주고받은 뒤, 받는 쪽에서 다시 날짜 객체로 변환해야 합니다.
