# ECMAScript (JavaScript)

[ECMA-262](https://tc39.es/ecma262/)는 우리가 흔히 '자바스크립트'라고 부르는 언어의 **공식 명세(Specification)**입니다. Chrome의 V8 엔진, Safari의 JavaScriptCore 등 모든 JS 엔진은 이 문서를 기준으로 만들어집니다.

---

## 1. 왜 표준을 알아야 할까요?

*   **"이 코드가 왜 브라우저마다 다르게 동작하지?"**: 표준이 불명확했던 과거(IE 시절)와 달리, 최신 브라우저들은 이 표준을 매우 엄격하게 따릅니다.
*   **"신기능은 언제 쓸 수 있지?"**: 매년 6월에 새로운 버전(ES2024, ES2025...)이 확정됩니다. `Array.prototype.at()` 같은 편리한 기능도 이 표준에 먼저 등재된 후 브라우저에 배포됩니다.

---

## 2. 자바스크립트의 본질

이 문서는 자바스크립트를 **"프로토타입 기반의 객체 지향 언어"**라고 정의합니다.

### Primitive vs Object
자바스크립트의 모든 데이터는 둘 중 하나입니다.
1.  **Primitive (원시값)**: `undefined`, `null`, `boolean`, `string`, `symbol`, `number`, `bigint`. (불변값)
2.  **Object (객체)**: 위 7가지를 제외한 나머지 모든 것 (함수, 배열 포함).

---

## 3. 개발자가 자주 마주치는 표준 동작

### 자동 세미콜론 삽입 (ASI)
자바스크립트는 줄 바꿈을 하면 자동으로 세미콜론이 있는 것처럼 처리해줍니다.
하지만 `return` 문 바로 다음에 줄을 바꾸면, 의도치 않게 `undefined`가 반환되는 함정이 있습니다.

```javascript
// 의도
return
{
  name: "kim"
}

// 실제 동작 (표준)
return; // 여기서 끝남!
{
  name: "kim"
}
```

### 부동소수점 (IEEE 754)
`0.1 + 0.2 !== 0.3` 문제는 자바스크립트 버그가 아니라, ECMA-262가 숫자를 처리할 때 **IEEE 754** 표준을 따르기로 정했기 때문입니다.
