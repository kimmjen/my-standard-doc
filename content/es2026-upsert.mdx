# Map.prototype.upsert

[Map.prototype.upsert](https://github.com/tc39/proposal-upsert)는 ECMAScript 2026에서 Stage 4로 확정된 제안으로, Map에서 **키의 존재 여부에 따라 삽입 또는 업데이트를 단일 호출**로 처리합니다.

---

## 1. 기존 문제점

Map에서 "있으면 업데이트, 없으면 삽입"을 하려면 매번 이런 패턴을 써야 했습니다:

```javascript
// 기존 방식: 조회 + 조건 분기
if (map.has(key)) {
  map.set(key, map.get(key) + 1);
} else {
  map.set(key, 1);
}
```

이 패턴은 `has` -> `get` -> `set`으로 3번 조회가 필요하고, 반복적이며 실수하기 쉽습니다.

---

## 2. upsert 사용법

```javascript
// 새로운 방식: 한 번의 호출로 처리
map.upsert(key, {
  insert: () => 1,           // 키가 없을 때: 초기값 1 삽입
  update: (existing) => existing + 1  // 키가 있을 때: 기존 값 + 1
});
```

### handler 옵션

| 옵션 | 호출 시점 | 인자 | 반환값 |
|------|----------|------|--------|
| `insert` | 키가 존재하지 않을 때 | 없음 | 삽입할 값 |
| `update` | 키가 이미 존재할 때 | 기존 값 | 업데이트할 값 |

둘 중 하나만 제공할 수도 있습니다:

```javascript
// insert만: 없을 때만 삽입 (이미 있으면 무시)
map.upsert(key, { insert: () => defaultValue });

// update만: 있을 때만 업데이트 (없으면 무시)
map.upsert(key, { update: (v) => v + 1 });
```

---

## 3. 실용 예시

### 단어 빈도 카운터

```javascript
const wordCount = new Map();

for (const word of words) {
  wordCount.upsert(word, {
    insert: () => 1,
    update: (count) => count + 1,
  });
}
```

### 캐시 패턴

```javascript
const cache = new Map();

function getOrCompute(key, computeFn) {
  return cache.upsert(key, {
    insert: () => computeFn(key),
  });
}
```

---

## 4. 반환값

`upsert`는 항상 **최종 저장된 값**을 반환합니다:
- `insert`가 호출되었으면 `insert()`의 반환값
- `update`가 호출되었으면 `update(existing)`의 반환값
