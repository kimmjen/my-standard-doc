# WCAG 3.0

[WCAG 3.0](https://www.w3.org/TR/wcag-3.0/)은 W3C의 **차세대 웹 접근성 가이드라인**입니다. 기존 WCAG 2.x의 구조적 한계를 극복하고, 더 넓은 범위의 사용자 요구를 반영하기 위해 설계되었습니다.

---

## 1. WCAG 2.x vs 3.0

| 구분 | WCAG 2.x | WCAG 3.0 |
|------|----------|----------|
| 구조 | 원칙 > 지침 > 성공 기준 | 가이드라인 > 결과(Outcomes) |
| 적합성 등급 | A, AA, AAA | Bronze, Silver, Gold |
| 평가 방식 | 합격/불합격 (pass/fail) | 점수 기반 (scoring) |
| 적용 범위 | 웹 콘텐츠 중심 | 웹, 앱, 문서, XR, IoT 등 |
| 기술 변화 대응 | 느림 (버전 단위) | 빠름 (모듈식 업데이트) |

---

## 2. 핵심 변경 사항

### 점수 기반 평가

WCAG 2.x의 이진(pass/fail) 평가 대신, **연속적인 점수 체계**를 도입합니다. 완벽하지 않더라도 부분적으로 접근성을 달성한 정도를 측정할 수 있습니다.

### 적합성 등급 변경

- **Bronze**: 기본적인 접근성 충족 (WCAG 2.x AA에 대응)
- **Silver**: 향상된 접근성
- **Gold**: 최고 수준의 접근성

### 확장된 적용 범위

웹 페이지뿐 아니라 모바일 앱, PDF 문서, VR/AR 콘텐츠, IoT 기기 인터페이스까지 포함합니다.

---

## 3. 가이드라인 구조

WCAG 3.0의 가이드라인은 다음 요소로 구성됩니다:

- **Guideline**: 접근성 목표를 서술적으로 정의
- **Outcome**: 달성해야 할 구체적 결과
- **Method**: 결과를 달성하기 위한 기술별 구현 방법
- **How-to**: 실무자를 위한 단계별 가이드
- **Test**: 결과 충족 여부를 평가하는 테스트 절차

---

## 4. 개발자에게 미치는 영향

### 색상 대비

WCAG 2.x의 단순 대비율(4.5:1, 3:1) 대신 **APCA(Advanced Perceptual Contrast Algorithm)**를 도입합니다. 폰트 크기, 굵기, 배경색을 종합적으로 고려하여 더 정확한 가독성을 측정합니다.

### 인지 접근성 강화

기존에 미약했던 인지 장애 사용자를 위한 가이드라인이 대폭 강화됩니다:
- 명확한 언어 사용
- 일관된 네비게이션
- 오류 방지 및 복구
- 주의력 관리

---

## 5. 현재 상태

WCAG 3.0은 2026년 3월 기준 **Working Draft** 단계입니다. 최종 권고안(Recommendation)까지는 추가 시간이 필요하지만, 새 프로젝트에서는 WCAG 3.0의 방향성을 미리 고려하는 것이 좋습니다.

기존 WCAG 2.x 기준을 충족하는 것은 여전히 유효하며, WCAG 3.0은 이를 대체하되 점진적으로 전환될 예정입니다.
