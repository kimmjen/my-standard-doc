# Iterator Sequencing

[Iterator Sequencing](https://github.com/tc39/proposal-iterator-sequencing)은 ECMAScript 2026에 포함된 제안으로, **여러 이터레이터를 하나로 연결**하여 순차적으로 소비할 수 있게 합니다.

---

## 1. 기존 문제점

여러 이터레이터를 순서대로 처리하려면 직접 연결 로직을 작성해야 했습니다:

```javascript
// 기존: 배열로 변환 후 합치기 (메모리 낭비)
const combined = [...iter1, ...iter2, ...iter3];

// 또는 제너레이터로 직접 연결
function* concat(...iterables) {
  for (const it of iterables) yield* it;
}
```

첫 번째 방식은 모든 값을 메모리에 올리고, 두 번째는 매번 헬퍼를 정의해야 합니다.

---

## 2. Iterator.concat()

새로운 정적 메서드 `Iterator.concat()`으로 이터레이터들을 지연(lazy) 연결합니다:

```javascript
const combined = Iterator.concat(iter1, iter2, iter3);

for (const value of combined) {
  console.log(value); // iter1 -> iter2 -> iter3 순서대로 소비
}
```

### 특징

- **지연 평가(Lazy)**: 값이 필요할 때만 다음 이터레이터에서 가져옴
- **메모리 효율적**: 전체를 배열로 변환하지 않음
- **Iterable 프로토콜 준수**: `for...of`, 스프레드 연산자 등과 호환

---

## 3. 실용 예시

### 페이지네이션된 API 결과 합치기

```javascript
async function* fetchPage(page) {
  const res = await fetch(`/api/items?page=${page}`);
  const data = await res.json();
  yield* data.items;
}

// 여러 페이지를 하나의 스트림으로
const allItems = Iterator.concat(
  fetchPage(1),
  fetchPage(2),
  fetchPage(3)
);
```

### 여러 소스의 로그 병합

```javascript
const appLogs = readLogFile("app.log");
const errorLogs = readLogFile("error.log");

const allLogs = Iterator.concat(appLogs, errorLogs);
```

---

## 4. Iterator Helpers와의 관계

ES2025의 Iterator Helpers(`map`, `filter`, `take` 등)와 자연스럽게 결합됩니다:

```javascript
const result = Iterator.concat(iter1, iter2)
  .filter(x => x > 0)
  .map(x => x * 2)
  .take(10);
```
