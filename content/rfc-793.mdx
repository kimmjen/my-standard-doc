# TCP (Transmission Control Protocol)

[RFC 793](https://datatracker.ietf.org/doc/html/rfc793)은 인터넷의 신뢰성을 책임지는 전송 계층 프로토콜인 **TCP**를 정의합니다.
우리가 보내는 데이터가 **"순서대로"**, **"빠짐없이"** 도착한다는 것을 보장해주는 고마운 기술입니다.

---

## 1. 핵심 메커니즘: 3-Way Handshake
TCP 연결을 맺을 때는 반드시 3단계의 인사를 나눕니다.
이 과정이 있어야 양쪽 다 데이터를 주고받을 준비가 되었는지 확인할 수 있습니다.

1.  **SYN (Synchronize)**: "안녕? 나 너랑 대화하고 싶어." (클라이언트 -> 서버)
2.  **SYN-ACK**: "그래, 확인했어. 나도 준비됐어." (서버 -> 클라이언트)
3.  **ACK (Acknowledge)**: "오케이, 연결 시작하자!" (클라이언트 -> 서버)

> [!TIP]
> **왜 중요할까요?**
> 서버가 다운되거나 네트워크가 끊겼을 때, 애플리케이션이 "연결 타임아웃(Connection Timeout)"을 낼 수 있는 근거가 바로 이 과정에서 응답이 오지 않기 때문입니다.

---

## 2. 흐름 제어와 혼잡 제어
TCP가 단순한 연결 도구가 아니라 '지능적인' 프로토콜인 이유입니다.

*   **Flow Control (흐름 제어)**: 받는 사람이 처리할 수 있을 만큼만 데이터를 보냅니다. (상대방의 버퍼 크기 고려)
*   **Congestion Control (혼잡 제어)**: 네트워크 망 자체가 막히면 전송 속도를 스스로 줄입니다. (Slow Start 알고리즘 등)

---

## 3. TCP vs UDP
개발자가 가장 자주 하는 고민 중 하나입니다.

| 특징 | TCP | UDP (User Datagram Protocol) |
| :--- | :--- | :--- |
| **신뢰성** | 높음 (전송 보장) | 낮음 (유실될 수 있음) |
| **속도** | 상대적으로 느림 | 매우 빠름 |
| **연결** | 연결 지향 (Connection-oriented) | 비연결형 (Connection-less) |
| **사용처** | 웹(HTTP), 이메일, 파일 전송 | 실시간 스트리밍, 온라인 게임, DNS |
